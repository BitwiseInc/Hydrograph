Aggregate Operation Class Window

Operation class defines the operations that can performed on the input data using the aggregate component. Operation class is a mandatory attribute. Operation class can be:

  • In-built operation class
  • Custom built operation class

Fig 1

In-built Operation Class

Hydrograph provides in-built set of functions in form of java classes that can be applied to the component. Aggregate component has below mentioned java classes:

  • Sum: Calculate the sum of a specified field within the group.
  • Max: The max function calculates the maximum value for the specified field within the group.
  • Min: The min function calculates the minimum value for the specified field within the group.
  • Count: The count function calculates the count of records within a group.
  • StringAppend: The string append function appends all the values of the specified field within the group.

The in-built operation class can be selected from the operation class drop down.

Custom Operation Class

User can create new custom classes or use the already available classes in the workspace.

Creating new custom class

To create a new custom operation class, select "Custom" in the operation class drop down and click on "Create New". The "New Java Class" window will open.

Fig 3

In the "New Java Class" window, click ‘Browse’ to select ‘Source Folder’. Enter or browse the package name and the class name for the new class and click "Finish". Upon successful creation of the new class, the control will be redirected to operation class window. The "Open" button will open the java class to edit it. Alternatively, the new class can also be opened by browsing it from the "Project Explorer".

Note: All custom operation classes for aggregate component must implement AggregateTransformBase interface.

Using existing custom class

Alternatively, instead of creating a new java class, an existing java class, compatible with the aggregate component can also be used. To use an existing java class, click "Browse" button and select the class available in the workspace. The class must implement AggregateTransformBase interface.

Operation Properties Grid

The operation properties grid is present on the operation class window. This grid is used to define operation properties. Every row added to this grid defines a new operation property. A property has a unique name and a value. The property value can also be parameterized by using the parameter syntax (i.e. using @{}).

These property values can be accessed in the operation classes.

Operation property may or may not be specified. It is not mandatory.
