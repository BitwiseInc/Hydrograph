Join Properties

Properties or the Join component can be viewed by Double click->component on canvas. The properties contain a 'General' tab and a 'Schema' tab. Common and mandatory properties are present in the General tab. Schema displays option to accept the field schema i.e. field name, Data type, scale etc.

General Properties

Display

  • Name - The identifier for the component. This is a mandatory property. This property is pre-populated with the component name, i.e. 'Join' followed by an incremental number. It can be changed to any custom name. The name property has following restrictions:
    • Must be specified and should not be blank.
    • Must be unique across the job.
    • Accepts only alphabets (a-z), numerals (0-9) and 4 special characters: "_", "-", ",", " " (space)<./span>
  • ID - ID field will specify unique id for every component.
  • Type - Type defines the type of component within the category. This typically is the name of the component. This is a non editable field.

Configuration

  • Input Count It accepts numeric values that display the number of input components attached to the Join component. By default this value is 2. On increasing the number of Input count the Join component transforms showing number of ports equal to the Input count.

  • Join Configuration -- Edit opens a grid that accepts Join keys for each input port. Along with it user can specify the type of Join : Inner or Outer. Join Type can also be parameterized which will be resolved during runtime.

  • Join Mapping - Edit option opens up the Join mapping view, where the user can select the input fields from each input needed for the Join output. Click on Mapping View to understand the Mapping view for Join component in detail

  • Runtime Properties - Runtime properties are used to override the Hadoop configurations specific to this join component at run time. User is required to enter the Property Name and Property Value in the runtime properties grid.

    Check How to pass Hadoop properties to component

  • Batch - Batch accepts an integer value and signifies the batch this component will execute in. The default value for batch is 0. Batch can have a maximum value of 99. Batch is a mandatory property.

Schema Tab

Schema is mandatory for join component. Schema tab defines the record format on the out port of the join component. A field in schema has multiple attributes as described below.

  • Field Name - The name for the field. This is a mandatory attribute.
  • Data type - The data type for the field. This is a mandatory attribute. The default data type is "String". Check supported data types page for list of supported data types.
  • Scale - The number of digits to the right of decimal point. Scale is defined for Double, Float or BigDecimal field.
  • Scale Type – Scale Type accepts values as implicit or explicit for BigDecimal field and none for other data types. Explicit considers the length of '.' in precision and implicit ignores length of '.' precision for the BigDecimal field.
  • Date Format - The format for date data type. Refer to Date formats page for acceptable date formats.
  • Precision – The number of significant digits (all digits except leading zeros and trailing zeros after decimal point).
  • Field Description – The description for the field.

Pull Schema – The schema defined in operation editor's output, will be pulled to the schema tab. The current schema in the grid will be overwritten with the schema from operation editor's output.

Validations

The join components applies validations to the mandatory fields as described above. Upon placing the join component on job canvas for the first time (from component palette), the component shows up a warning icon as mandatory properties are not provided.

The properties window also displays error icon on mandatory fields if it has an incorrect value. The error icon is displayed on the tab as well, if any of the field within the tab has some error.

If the properties window has some error even after user visit's it once, then the warning icon on the join component on the job canvas changes to error icon. This error icon is removed only when all the mandatory fields are supplied with correct values.
