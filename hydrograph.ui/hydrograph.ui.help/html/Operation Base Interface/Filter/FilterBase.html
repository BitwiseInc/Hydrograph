FilterBase

The FilterBase interface is the base interface for all the custom classes defined for filter in Filter component. This interface exposes methods that enable users to define custom filter expressions.

Method Detail

isRemove

public void isRemove(ReusableRow inputRow)

This method isRemove()is the operation function and is called for each input row.The custom filtering logic should be performed in this function. This function returns a boolean value which determines if the record is to be excluded from the output.

If the function returns true the record is rejected from the output, i.e.transmitted to the unused port of the filter component.

If the function returns false the record is retained in the output, i.e.transmitted to the out port of the filter component.

The parameter input holds the current input row. Refer the documentation of ReusableRow class to get information on accessing individual field of the current input row.

cleanup

public void cleanup()

The method cleanup() is called after processing all the records in the input. This method can be typically used to do cleanup activities as the name suggests.

prepare

public void prepare(Properties props, ArrayList inputFields)

The method prepare() is called before processing the first record in the input. This method can be used for initialization / instansiation of artifacts to be used in the filter expressions.

Parameters

props
The props object holds the operation properties supplied to the filter component using operation properties grid on operation class window. The individual properties are wrapped as key-value pairs in the props object.

inputFields
The inputFields holds the list of input fields to the filter operation.
